## 1. タスクフォースの目的・コンセプト(p.1–2)

- **コンセプト**: Googleの最新AIエージェント技術 × 自社のデータ活用ノウハウを統合し、自律的にタスクを遂行するAIエージェントを構築。「自社の生産性向上」と「構築ノウハウの事業化」を同時推進。
- **背景**: GoogleがGemini Enterprise Agent Platformを大々的にリリース。今後AIエージェントによる業務効率化が多くの企業で求められる。
- **位置づけ**: 自社を **"クライアントゼロ"** とし、蓄積した応用事例をパッケージ化。
- **目的(2軸)**:
  1. GWS, Hubspot, Backlog 等の**自社データ**を活用したAIエージェントで社内生産性向上
  1. GA4, BQ 等の**顧客データ分析環境**でのAI活用ノウハウをパッケージ化し、大企業向けAI導入支援事業を創出

## 2. 体制(p.3 初期 / p.14 拡張後)

- **意思決定・アドバイザリー**
  - スポンサー: 甲斐・三田村(最終承認と経営資源の最適化)
  - プロジェクトオーナー: 海老澤(進捗・成果の総括、部門間調整)
  - アドバイザー: 野口・田中(亨)(専門的知見に基づく助言、リスク検知)
- **統括・事務**
  - 全体統括: 西本・渡邊 / 技術主幹: 川村・菊地(技術選定・アーキテクチャ設計・**セキュリティ**)/ 事務局: 平松・今村
- **分科会構成**: 初期は3分科会(ガバナンス / 社内利用促進 / 外販)→ 後にテーマ別へ細分化(p.14)。
  - ガバナンス分科会の初期スコープ = **内部統制・インフラ構築**

## 5. 課題・留意事項(p.9)— ガバナンス観点の明示事項

- **担当者アサイン**: 部課長が分科会メンバーの既存業務稼働を調整。進捗を統合管理し分科会間連携を促す。
- **環境構築**: 分科会が勝手にインフラ構築しないよう注意(=シャドーIT防止)。禁止事項・利用制限は速やかにメンバーへ共有。
- **技術課題**:
  - 顧客データ・機密情報の取り扱いには **GCPのIAM権限設定を厳格に適用**
  - **グラウンディング技術**でハルシネーションを抑制し信頼性を担保

## 6. 参考:目指すアーキテクチャ(p.11, Google Cloud Next 出典)

エージェント基盤の参照構成。**当チームのハーネス(技術統制)設計の起点**となる:

- Frontend → **Agent Gateway** → **Agent Identity (Agent Platform Runtime)** → Agent Gateway → Agents/Tools/Models/APIs
- 上段の横断統制レイヤ: **Agent Registry / AI Security / Access Authorization**
- 下段: **AI Observability**(全体の可観測性)
- Agent内部: Profile・goals・skills・instructions・tools / model-based reasoning & task execution loop / Memory & Sessions / Models(Gemini/3P)

> 監視・統制すべき技術ポイント(エージェントレジストリ、AIセキュリティ、アクセス認可、可観測性)が公式アーキ図に対応している。ハーネス項目はこの図にマッピングして網羅性を担保できる。

## 7. 参考:AI導入・運用支援メニュー(p.10)

「Gemini × Google Workspace 実務直結型研修」。人材像(全社員/活用者/推進者・開発者/経営層・管理職)× レベル(Lv.1 知る 〜 Lv.4 創る・変革する)の学習ロードマップ。Lv.1に「**AIリスク・セキュリティ・プライバシー**入門コース」「生成AI活用における法律・リスク(セキュリティ/プライバシー)」が含まれる(=ガバナンス啓発の既存資産)。

## チームへの示唆(要約者メモ)

1. **チームの公式スコープ = p.6-7「内部統制」テーマ**: 機密データフィルタリング / プロンプト監査ログ監視フロー / AI生成物の著作権・法的適合性。まずこの3点を具体化する。
1. **マイルストーンは F2「ガバナンス策定」(2026/6–7)**: 利用指針(禁止事項)・法規制照合・事故対応方針の明文化が期限つきで求められている。← 最初の成果物はここに照準。
1. **ハーネスの参照アーキは p.11**: Agent Registry / AI Security / Access Authorization / AI Observability に監視項目をマッピングすると網羅性の根拠になる。
1. **「社内版」と「外販テンプレ版」の二形態**を意識: 外販準備分科会と連携し、成果物をそのまま顧客提供パッケージへ転用。
1. 既出のガバナンス断片(GCP IAM厳格化、グラウンディング、シャドーIT防止、研修Lv.1のリスク講座)を体系の中に位置づけ直すと、既存施策との整合が取れる。
