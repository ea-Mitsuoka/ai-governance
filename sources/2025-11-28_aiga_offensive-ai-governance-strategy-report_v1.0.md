## 0. 本レポートの「AIガバナンス」の射程(p.3–4)

- 「AIガバナンス」を、**経営・取締役会レベルの意思決定**と、**適切なリスク管理を通じてAIの価値を最大化する取組**全般を包括的に指す語として使用。
- AI事業者ガイドライン(第1.1版)の **アジャイル・ガバナンス**(事前固定ルールでなく『環境・リスク分析→ゴール設定→システムデザイン→運用→評価』をマルチステークホルダーで高速に回す実践)を望ましいモデルとして引用。
- **外ループ(経営/コーポレート)** と **内ループ(現場)** の双方+その連携を「ガバナンス」と総称(下図思想)。
  - 外ループ=経営のコミットメントと責任明確化、ルール・組織体制の評価改善
  - 内ループ=個別AIシステムのリスク・影響を事前確認し対策を実装

## 1. AIの正負インパクトとリスク類型(p.5–8)

- 正のインパクト: 生成AIで世界経済に2.6–4.4兆ドルの価値創出可能との試算。価値の約75%は顧客対応・マーケ営業・ソフト開発・研究開発に集中。
- **「AIを活用しないことのリスク」** を政府文書(AI事業者ガイドライン別添 p.22、金融庁AIディスカッションペーパー)が明言 → 過度な安全志向で動けないこと自体がリスク。
- **AIリスクの2分類**:
  - **事業上のリスク**: 売上・コスト等の経営指標に直接影響(品質、ハルシネーション→株価毀損例=1000億ドル、悪用、海外LLM依存=事業継続性/サプライチェーン)。
  - **社会的なリスク**: 直接の経営指標には響かないが、市民・政府・投資家を通じブランド/企業価値を毀損(プライバシー、偏見・差別、偽情報)。
- 自社が「AIを開発するのか/他社AIを使うのか」で想定リスク・統制が変わる(開発者・提供者・利用者の区分=AI事業者ガイドライン準拠)。
- 資本市場の圧力: SECがAIウォッシング(AI性能の誇張)を処分、株主提案・デューデリフレームワークの登場。「正しい理解だけでなく適切な統制措置」が当然視される時代へ。

## 2. 日本企業の課題:方針の不在とスモールスタートの限界(p.9–11)

- 生成AIを「積極活用方針」と回答した企業: 中国48.5% / 米独39.2% / **日本23.7%**。
- 業務での生成AI利用率: 中国95.8% / 米90.6% / 独90.3% / **日本55.2%**。
- 日本はリスク選好度が低く不確実性下の意思決定を避けがち。「リスクを過大評価して進められない」or「対策過剰でROIが出ず頓挫」の二極。
- スモールスタートは現場が着手しやすい一方、**経営目標・戦略がないと全社展開できない** → トップダウンの全社戦略が必要。

## 3. ★中核:経営意思決定フレームワーク試案(p.12–18)

### 3.1 AI時代の経営に求められる基本姿勢(3点)

1. 基礎的な**AIリテラシー**を身につける
2. **AIと企業価値のつながりを理解**し、経営層の誰もが自分ごととして向き合う
3. AI活用・ガバナンス推進を\*\*「大規模な企業変革=チェンジマネジメント」\*\*としてとらえ、経営意思決定の質とスピード向上を最終目的に推進

### 3.2 意思決定フレームワーク(Step0–4)

- **Step0 「あるべき姿」の設定**: ミッション・ビジョンと整合した北極星。国際的に合意されたAI原則を参照。
- **Step1 自社AI棚卸し**: 「誰が・どの業務で・どんなAIを・どう使っているか」を網羅調査(**シャドーAI含む**)。「いつの間にかAI」リスクの芽を解消し現在地を把握。
- **Step2 活用可能性の探索+初期スクリーニング**: 機会の発見(外部環境×内部環境)と網羅性(競争/顧客/ブランド/事業モデル/人材・文化の5観点)。有望案にリスクシナリオを特定し、戦略的重要性・顕在実績・潜在インパクト・自社関連性の4軸で優先順位づけ。
- **Step3 ベネフィット/リスクの分析評価(AI-RA-ROICモデル)**: 後述。
- **Step4 アジャイル・ガバナンスによる継続更新**: ゴール設定自体も環境変化に応じ見直し。外ループ⇄内ループの連動が要。

### ★ AI-RA-ROIC モデル(p.15–18)— 定量評価の核

```
AI-RA-ROIC(AIリスク調整後ROIC) = (NOPAT − RAAC) / (投下資本 + 追加対策コスト)
  NOPAT = 営業利益 ×(1 − 実効税率)
  RAAC(リスク調整後AIコスト)= 損失の発生確率(LEF)× 損失の大きさ(LM)  ← 年間期待損失額
```

- サイバーセキュリティの **(RA-ROIC)** と金銭換算の国際標準 **FAIR(Factor Analysis of Information Risk)** を土台に構築。
- 算出手順: ①ベースラインROIC(便益のみ)→ ②AIリスクをRAAC(=LEF×LM)で定量化 → ③追加対策コスト(セキュリティ費+業務再設計・再教育の「変革コスト」)とリスク低減効果を見積り → ④AI-RA-ROIC算出 → ⑤**Go/No-Go判定**(ハードレート・現状維持ROICと比較し、残留リスクRAACが事前定義の**リスク許容度**内か)。
- No-Go案も、活用形態をアップデートして再検討する出発点と位置づける。
- **留意**: AIリスクはブラックボックス性・倫理的社会的影響など独自性があり、既存IT/サイバー枠組みの流用には限界。社会的リスクの定量化は今後の課題と明記。

## 4. AIガバナンスエコシステム形成の課題(p.19–21)

- **取締役会の責任として確立**: 会社法の内部統制システム構築義務の一内容として**AIガバナンス体制構築義務**が認められるとの見解(日本銀行金融研究所「金融機関におけるAIの利用を巡る法律問題研究会」)。取締役はAI利用の組織内の最終責任を負う。
- **開示のコンセンサス構築**: 「コンプライ・オア・エクスプレイン」では形式的コンプライは効果薄。**独自モデルを構築し対外的にエクスプレインしてトラスト確保**へ。広島AIプロセス(HAIP)の報告枠組みが参考。
- **公的枠組みでの機会・リスク評価**: 政府機関・AIGA等がファシリテーション。AIガバナンスナビ等の既存ツール活用。

## 自社(e-Agency / 内部統制分科会)への適用示唆

1. **「攻めのガバナンス」フレーミングは外販の強力な売り**: 顧客(大企業の経営層)に響くのは「守り」より「リスクテイクの意思決定支援」。外販テンプレートの上位コンセプトに採用したい。
2. **Step1「自社AI棚卸し(シャドーAI含む)」はS0/S3で即着手すべき**: 体制草案p.9「勝手にインフラ構築しない」と直結。棚卸しは当チームのリスク評価の基礎インプット。**最初の具体タスク候補**。
3. **AI-RA-ROICは「内部統制」テーマの定量的背骨になりうる**: 起点①機密データ②監査ログ の対策コストを RAAC低減効果として説明でき、経営に「なぜこの統制に投資するか」を数字で示せる。PoC(F3)のリスク評価・Go/No-Go判定にそのまま使える。
4. **会社法・内部統制システム構築義務との接続**: 当チームの活動を「取締役の善管注意義務/内部統制義務の履行」と位置づけると、経営層の承認(スポンサー:甲斐・三田村)を得やすい。J-SOX文脈の根拠になる。
5. **アジャイル・ガバナンス(外⇄内ループ)はロードマップS4「改訂ループ」の理論的裏付け**: 当チームの改訂自動化はこの概念の実装と説明できる。

## 起点3点との関連

- **① 機密データフィルタリング**: Step1棚卸しで「関連データ資産」を特定 → リスクシナリオ(ⓑシャドーAI/不適切入力による機密・知財漏洩, 脚注49)の対象に。RAACのLM(損害の大きさ)算定対象。
- **② プロンプト監査ログの監視フロー**: 内ループ(現場の評価・対策実装)と外ループ(経営の体制評価)の連動を支える証跡。Step4の継続更新の情報源。
- **③ AI生成物の著作権・法的適合性**: リスクシナリオ(ⓓ不適切情報開示・AIウォッシングによる法的・レピュテーションリスク, 脚注49)に対応。社会的リスク類型に該当。
