## 1. ISO/IEC 42001とは(AIMS)

- **世界初のAIマネジメントシステム(AIMS)国際標準**(2023年12月発行)。ISO 9001(品質)・ISO 27001(情報セキュリティ)と同じ**マネジメントシステム規格(MS規格)**=PDCAで継続改善し、**認定された第三者認証機関による認証(certification)が取得できる**。
- **コア構成要素**:
  - AIMS(他のISO標準と統合し継続改善)
  - **AIリスクアセスメント**(ライフサイクル全体のリスク特定・低減)
  - **AIインパクトアセスメント**(個人・社会への影響評価=リスクアセスメントと別概念)
  - データ保護とAIセキュリティ
- **認証が組織にもたらすもの**: 透明・信頼・倫理的なAI/EU AI Act等のコンプライアンス対応/リスク管理と説明責任の向上/顧客・利害関係者の信頼/AIガバナンスと事業目標の整合。

## 2. ★ Statement of Applicability(SoA・適用宣言書)

- **ISO/IEC 42001は、特定されたすべてのAIリスクを正式に文書化し、具体的な統制目標(control objectives)に紐づけることを要求**。これを **SoA(Statement of Applicability)** にまとめる=認証の重要構成要素。
- → **これは本プロジェクトの[リスク台帳(A-2 ASI×T)+対策マッピング](../docs/to-be/01-risk-classification-and-grading.md)とほぼ同じ構造**。ISOはそれを認証成果物として形式化している。

## 3. 周辺標準のエコシステム

ISO/IEC 42001を支える補助標準(認証必須ではないが実装の道具):

| 標準 | 役割 |
|---|---|
| **ISO/IEC 23894** | AIリスク管理のガイダンス(フレーム+プロセス)。※ロードマップS1必須の一つ |
| ISO/IEC TR 24027 | AIのバイアス・公平性の評価と低減 |
| ISO/IEC TR 24028 | AIの信頼性(trustworthiness)概観 |
| ISO/IEC TR 24029-1 | ニューラルネットのロバストネス評価 |
| ISO/IEC 5259(-1) | 機械学習のデータ品質 |
| ISO/IEC 22989 | AI用語・基本概念(語彙) |
| ISO/IEC 23053 | MLを用いたAIシステムのフレームワーク |
| ISO/IEC TR 24368 | AIの倫理的・社会的懸念 |
| ISO/IEC 38507 | 統治機関(ガバナンスボディ)のAI監督 |
| ISO/IEC 27001 | ISMS(情報セキュリティ)。**42001はこれと整合するよう設計**=既存ISMSに接ぎ木可 |
| ISO/IEC 31000 / Guide 73 | 汎用リスク管理フレーム・リスク用語 |

> ISO 42001は **NIST AI RMF・OECD・EU AI Act** とも整合し、「責任あるAIのグローバル基盤」を形成、と位置づけ。

## 4. 認証監査のプロセス(参考)

- **多段階・3年サイクル**: Pre-Audit(事前監査)→ 初回認証監査【Stage I=文書監査 / Stage II=実装監査】→ Post-Audit(是正)→ 1年目・2年目のサーベイランス監査 → 3年目に再認証。
- **6つの監査フェーズ**: ①スコープ定義 ②リスクアセスメント ③適切な文書化(Stage I)④実装・運用監査(Stage II)⑤是正・改善のPost-Audit ⑥決定通知・認証。
- **認証は一度きりでない**=継続的コミットメント(年次サーベイランスで継続改善・統制目標の有効性を確認)。

## 自社(e-Agency / 内部統制分科会)への適用示唆

1. **「第三者認証が取れる」という新しい柱**: NIST(任意)・OWASP(脅威)・国内GL(指針)と違い、ISO 42001は**認証取得=対外的な信頼の証明**になる。e-Agencyにとって(a)将来の自社認証目標(クライアントゼロの成熟度の証明)(b)\*\*外販で「顧客のAIMS構築・ISO 42001認証取得を支援」\*\*という強い訴求点、の二つの意味を持つ。
1. **SoA=うちのリスク台帳の到達点**: 「全AIリスクを文書化し統制目標に紐づける」SoAは、[A-2のASI×T台帳](../docs/to-be/01-risk-classification-and-grading.md)+対策(OWASP PB/NIST GV-MG)マッピングの**認証レベルの完成形**。台帳を作るときから「将来SoAに昇格できる粒度」を意識すると無駄がない。
1. **マネジメントシステムの三重裏付け**: ISO 42001(PDCA・MS)=[AI事業者GL第2部E](2026-03-31_meti-soumu_ai-business-guideline_v1.2_main.md)=[NIST GOVERN/MAP/MEASURE/MANAGE](2023-01_nist_ai-rmf-1.0-playbook.md)。三者が同じ「継続改善サイクル」=[改訂ループ・P8](../docs/to-be/00-principles-and-scope.md)の確度が最高レベルに。
1. **ISO 27001との統合=D-2の補強**: 42001はISMS(27001)と整合するよう設計。[D-2既存ポリシー基盤型](../docs/to-be/00-principles-and-scope.md)=「既存規程に接ぎ木」は、ISOの世界でも「既存ISMSにAIMSを統合」が定石=方針の正しさが裏付く。e-Agencyが27001を持つ/取るなら42001は載せやすい。
1. **AIインパクトアセスメントという別概念**: リスクアセスメント(組織のリスク)とは別に、**個人・社会への影響評価**を要求。[別添7C](2026-03-31_meti-soumu_ai-business-guideline_v1.2_worksheet.md)・[NIST AI 600-1](2024-07_nist_ai-rmf-generative-ai-profile_ai-600-1.md)の影響評価と接続。リスクと影響を分けて評価する設計をS3で検討。
1. **周辺標準は「必要に応じ参照」で十分**: 23894(リスク管理)以外のTR群(24027バイアス/24028信頼性/24029ロバストネス/5259データ品質等)は、特定領域を深掘りするときの参照リスト。今すぐ全部要らない。⚠規格本文は有償なので、認証を本気で狙う段階(S4以降)で取得を検討。

## 起点3点との関連

- 個別の技術論ではなく**マネジメントシステム(統治の器)**。起点3点の統制を「SoAに載る統制目標」として体系化する上位の枠。
- データ保護・AIセキュリティ(コア要素)→ 起点①。AIインパクト評価・継続監視 → 起点②。倫理・コンプライアンス・バイアス(TR 24027/24368)→ 起点③。
